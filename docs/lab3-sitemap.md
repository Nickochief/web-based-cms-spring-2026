## Justification

I broke the Employee Handbook into sub-pages that consisted of Benefits, Code of Conduct, and HR Policies. I did this because it is a bulky, multi-topic document that is hard to navigate as a single file. The target audience is internal employees that need to find specific information quickly without having to read the entire document.

I cut a lot of the marketing blurb because it is outdated and no longer accurate as it states it is from 2020. Since it has been years since it was made, the document could contain stale statistics, old brand messaging, and a company description that is no longer accurate to what the company is today.

I reorganized the server_config_notes.txt into three sub-pages, the first being Server Configuration, then API Settings & Feature Flags, and finally Known Issues & Workarounds because the original file was unformatted and it covered multiple distinct technical topics. There were also personal comments such as "TODO: Clean this up – Jeff" and "USE THIS EMAIL, NOT DAVE". Those were removed because they are unprofessional, and they are not necessary to have. There was also a dead link to the legacy login that was cut because it was outdated. This section is intended exclusively for engineers and developers.
